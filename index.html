Introduction

Vice was created as a response to the growing gap between system and scripting languages. When programming in C++, you gain performance but lose time managing memory and fighting complex syntax. Python, on the other hand, offers simplicity but hides how the computer actually processes data, often leading to performance bottlenecks.

Vice is the "Golden Mean". We designed it so that code reads like a manual. We introduced explicit data flow using arrows, allowing you to understand at a glance whether data is being sent to the user or retrieved from an external source. Built on Rust, Vice is safe and predictable, making it ideal for learning computer science and building rapid prototypes or 2D games.

Vice is a modern, high-level programming language designed for clarity and ease of use. It features a unique syntax for assignments and data flow, making it intuitive to read and write. Unlike many languages that hide complexity behind magical syntax, Vice makes every operation explicit — you always know where data comes from and where it goes.

✨ Key Features

Strongly Typed: Catch errors before you run the code. Vice's type system prevents entire categories of runtime bugs by validating your logic at parse time.

Built-in UI: Create windows and draw shapes with zero configuration. No external libraries, no dependency hell — just write ui.create() and you're done.

Clean Syntax: Uses arrows (=>, ->, <-) to clearly show where data is going. The direction of the arrow tells you everything about the data flow.

Hybrid Modes: Switch between @lowlevel and @highlevel in the same file. Write performance-critical code and safe business logic side by side.

🌱 For Beginners

Intuitive syntax without unnecessary boilerplate. Clear, human-readable error messages that explain what went wrong and where.

🎓 For CS Students

Explore the differences between integer and floating-point arithmetic within a single file. Understand how CPUs actually process numbers.

⚡ For Prototypers

Built-in UI engine lets you create interactive windows in 3 lines of code. Perfect for rapid prototyping and game jams.

🔒 For Safety-Critical Code

Immutable-by-default variables, exhaustive pattern matching, and strict type checking prevent bugs before they happen.

The heart of Vice is its attribute system. Imagine writing a banking system where every fraction of a cent matters — you use @highlevel to ensure maximum precision with floating-point math. If you're writing a particle engine for a game where every millisecond counts, you switch to @lowlevel for blazing-fast integer arithmetic. Vice is the only language that offers this flexibility within a single project, teaching conscious choices between performance and accuracy.

Project Goals

Vice aims to combine a clear syntax, typed data flow, and a lightweight runtime for small applications, educational use, and rapid prototyping.

1. 📖 Readability First: Code should look like instructions, not a math riddle. Every symbol in Vice has a clear, unambiguous meaning.

2. 🎓 Education: Vice explains errors in a human way, pinpointing the exact location and cause. Perfect for learning programming concepts without the frustration of cryptic compiler messages.

3. 🔀 Hybrid Ready: Preparing programmers to understand how numbers actually work in memory. The @lowlevel/@highlevel system teaches the real differences between integer and floating-point math.

4. 📦 Portable: A single executable file, zero heavy dependencies. Write once, run anywhere — Vice compiles to a standalone binary.

5. 🎮 Practical: Built-in UI engine for games and prototypes. No need to install SDL, SFML, or any external graphics library — Vice has everything you need.

Your First Program

In Vice, every program needs an entry point. This is the function that the computer runs first. We mark it with @entry. The entry function is always called main — this is a convention that Vice enforces to keep your code predictable and readable.

The output <- operator is Vice's way of sending data to the console. The arrow points left, meaning data flows from your program out to the user. This visual metaphor makes code instantly understandable: arrows always show the direction of data flow. 
                    

                        hello.vice
                        
                    

                    
                        @entry
                        fn main() {
                        output <- "Hello, Vice!";
                        output <- "Welcome to the future of programming.";
                    }
🔍 What's happening here?
• @entry — Tells Vice to start execution here
• fn main() — Defines a function named "main" with no parameters
• output <- — Sends the text to the console output
• The arrow <- shows data flowing OUT from your program

Variables & Constants

Variables are the heart of any program — they store the data your program works with. In Vice, we introduced a strict distinction between let (immutable) and mut (mutable), inspired by modern languages like Rust and Swift. Why is this important? Most programming errors occur when a value changes unexpectedly. By using let, you tell the compiler: "This value is constant, keep it safe." The compiler will refuse to compile if anything tries to modify it.

This might seem restrictive at first, but it's one of the most powerful features for writing correct code. When you see let, you know that value will never change — no need to trace through the code to find where it might be modified. When you see mut, you know to pay attention because this value can change over time. 
                

                    variables.vice
                    
                

                
                    let name : string => "Vice";        # Immutable — cannot be changed
                    mut count : int => 0;               # Mutable — can be updated
                    let ready : bool => true;           # Immutable boolean

                    count += 1;                          # ✅ Allowed: count is mut
                    # name = "NewName";                  # ❌ Error: name is let (immutable)

                    output <- "Counter:", count;

                    input -> raw_age;                    # Get input from user
                    let age => to_num(raw_age);          # Convert and store immutably
⚠️ Important Rule: In Vice, you must decide at declaration time whether a variable is mutable or immutable. This is not optional — it's a core language design principle. Defaulting to let (immutable) is recommended unless you specifically need to change the value later.
Keyword Mutability Usage
let Immutable Values that never change — constants, configuration, fixed data
mut Mutable Values that change — counters, accumulators, game state

Data Types

Vice is a strongly typed language. This means every variable has a specific type, and the compiler checks that you're using types correctly before your program runs. This catches an entire category of bugs — like trying to add a string to a number — at compile time rather than at runtime.

Vice 0.1.0 distinguishes between int (whole numbers) and float (decimal numbers). To a beginner, the difference between 5 and 5.0 might seem trivial, but for a CPU, they are completely different worlds. Integers are fast and exact; floats can represent fractions but introduce tiny rounding errors. Learning this distinction builds a solid foundation for understanding processor behavior. 
                

                    types.vice
                    
                

                
                    let name: string => "Alex";           # Text
                    let age: int => 25;                   # Whole number (i64)
                    let pi: float => 3.14159;             # Decimal number (f64)
                    let is_happy: bool => true;           # Boolean
                    let scores: int[] => [90, 85, 100];  # Array of integers
                    let names: string[] => ["A", "B"];   # Array of strings
Type Description Example Backing
int Whole number value 42, -7, 0 i64
float Decimal number value 3.14, -0.5, 2.0 f64
string Text value "Hello", "Vice" UTF-8 String
bool Boolean true/false true, false bool
Type[] Typed array [1, 2, 3] Vec<T>
💡 The Dot Rule: If you write 5, the compiler knows it's an int. If you write 5.0, it becomes a float. This simple rule makes type inference intuitive while teaching awareness of underlying data representations.

Input & Output

Vice makes data flow visual and intuitive. Instead of using function calls like print() or input(), Vice uses arrows to show the direction of data. This might seem like a small design choice, but it fundamentally changes how readable your code becomes — you can literally see data flowing through your program. 
                

                    io_demo.vice
                    
                

                
                    # Output: data flows OUT of your program to the user
                    output <- "The score is", score;
                    output <- "Hello", "World", "!";    # Multiple values supported

                    # Input: data flows IN from the user to your program
                    output <- "Enter your name:";
                    input -> user_name;
                    output <- "Hello", user_name;
Operator Direction Meaning
output <- Program → User Send data to console/output
input -> User → Program Receive data from user

The Hybrid System (@lowlevel vs @highlevel)

This is Vice's superpower. In most languages, the behavior of math operators is fixed. In Vice, you can decide how the runtime treats your logic by applying attributes to functions. This allows you to write high-performance "hot paths" and safe "business logic" in the same file.

Think about it: in a game, your rendering loop needs to be fast — integer math is perfect. But your inventory system needs to calculate prices with tax — you need floating-point precision. Vice lets you have both, in one language, in one file, without any weird workarounds.

🔀 Mode Comparison

5 / 2 result2 (integer division)2.5 (float division)
Float→IntImplicit truncationRequires explicit cast
PerformanceMaximum speedSlightly slower (safe checks)
Use caseGame loops, graphicsFinancial logic, science
                
hybrid_demo.vice

                
                    @lowlevel
                    fn fast_math() {
                    let result => 5 / 2;
                    output <- "LowLevel Result:", result;  # Output: 2 (integer division)
                }

                @highlevel
                fn precise_math() {
                let result => 5 / 2;
                output <- "HighLevel Result:", result; # Output: 2.5 (float division)
            }
🎯 Pro Tip: You can mix @lowlevel and @highlevel functions in the same file. Call one from the other — Vice handles the transitions automatically. This means you can write a @lowlevel physics engine that calls @highlevel scoring functions, and everything just works.

Type Inference & @autocheck

Writing let x: int => 5 can be tedious when the type is obvious from the value. That's why Vice features a type inference system activated by the @autocheck attribute. The semantic analyzer "looks" at the right side of the assignment and deduces the type for you. This results in cleaner code without sacrificing safety — once a type is assigned, it cannot change during execution.

We implemented the "Dot Rule": If you write 5, the compiler knows it's an int. If you write 5.0, it automatically becomes a float. This simple yet brilliant rule teaches awareness of underlying data types while offering the convenience of dynamic languages. 
            
inference.vice

            
                @autocheck
                @entry
                fn main() {
                let a => 5;         # Inferred as int (no decimal point)
                let b => 5.34;      # Inferred as float (has decimal point)
                let c => a + 10;    # Result is int (int + int = int)
                let d => "Hello";   # Inferred as string
                let e => true;      # Inferred as bool

                output <- a, b, c, d, e;
            }

Without @autocheck, Vice requires explicit type annotations for let and mut declarations. This ensures maximum clarity when you need it, and maximum convenience when you don't.

Pattern Matching

Traditional if-else statements are fine for simple conditions, but they become unreadable with complex logic. Pattern Matching (match) is a modern approach where you define patterns, and Vice executes only the code block that matches. It's not just about aesthetics; it's more efficient — the compiler can optimize these structures much better than a chain of independent conditions.

In Vice, match is an expression. This means the entire construct returns a value that you can assign directly to a variable. Note the _ symbol — it's a "wildcard" that catches all cases you didn't explicitly handle. This ensures your code is always exhaustive and resilient to unexpected input. 
            
match.vice

            
                @autocheck
                fn categorize(score: int) {
                let grade => match score {
                100 => "Perfect!",
                90 | 91 | 92 | 93 | 94 => "Excellent",
                80..89 => "Great",
                70..79 => "Good",
                60..69 => "Passed",
                _ => "Needs improvement"
            };
            output <- "Grade:", grade;
        }
📊 Results:
• score = 100 → Grade: Perfect!
• score = 91 → Grade: Excellent
• score = 85 → Grade: Great
• score = 50 → Grade: Needs improvement
💡 Pattern Matching Features:
• | — OR operator: matches any of the listed values
• .. — Range operator: matches any value in the range
• _ — Wildcard: matches everything (default case)
• The match expression must be exhaustive — all possible values must be covered

Arrows are the visual soul of Vice. Unlike most languages that use the "=" sign for everything, Vice distinguishes between initialization, printing data, and reading user input. This makes the "flow" of information visible at a glance. When you scan Vice code, you can instantly trace where data comes from and where it goes — no need to memorize function names or conventions.

Conditions & Logic

Control flow in Vice is built to be predictable. The if statement evaluates conditions that must result in a boolean value. Unlike some dynamic languages, Vice does not allow "truthy" or "falsy" values (like 0 or empty strings) to be treated as booleans. This strictness prevents a whole category of common logic bugs where unexpected type coercion leads to incorrect program behavior.

We support standard logical operators: && for AND, || for OR, and ! for NOT. This strict approach ensures that when you read a condition, you know exactly what it's checking without hidden type conversions. Every condition must evaluate to true or false — nothing else is accepted. 
            

                conditions.vice
                
            

            
                if (count > 10 && ready == true) {
                output <- "All systems go!";
            } else if (count > 5) {
            output <- "Almost there...";
        } else {
        output <- "Not ready yet.";
    }

    # Nested conditions
    if (score >= 90) {
    if (bonus_active) {
    output <- "Double reward!";
}
output <- "Excellent work!";
}
⚠️ Remember: In Vice, you cannot write if (count) to check if count is non-zero. You must write if (count != 0). This explicitness prevents subtle bugs and makes code intentions crystal clear.

Loops & Iteration

Loops allow your program to repeat actions — essential for processing arrays, animating objects, or running game loops. Vice provides three loop constructs: for loops with ranges, for loops with arrays (each loops), and while loops for condition-based repetition.

The range syntax 1..5 creates an inclusive range from 1 to 5. The for...in construct with arrays iterates over each element directly — no need to track indices manually unless you want to. 
            

                loops.vice
                
            

            
                # Range loop: iterates 1, 2, 3, 4, 5
                for i in 1..5 {
                output <- "Step:", i;
            }

            # Array loop: iterates over each element
            let nums => [10, 20, 30];
            for n in nums {
            output <- n;
        }

        # While loop: repeats while condition is true
        mut count => 3;
        while (count > 0) {
        output <- "Countdown:", count;
        count -= 1;
    }
    output <- "Liftoff! 🚀";
🔁 Loop Types Summary:
• for i in start..end — Range-based, inclusive
• for item in array — Element-based iteration
• while (condition) — Condition-based repetition
• All loops support break (exit early) and continue (skip to next iteration)

Functions

Functions are reusable blocks of code that perform specific tasks. They are the building blocks of any non-trivial program. In Vice, functions are defined with the fn keyword, can accept parameters with type annotations, and can return values using the -> return type syntax.

Functions can call other functions, creating a hierarchy of logic. This modular approach makes code easier to test, debug, and understand. Each function should do one thing well — if a function is doing too much, split it into smaller functions. 
            

                functions.vice
                
            

            
                # Simple function — no return value
                fn greet(name: string) {
                output <- "Hello,", name;
            }

            # Function with return type
            fn add(a: int, b: int) -> int {
            return a + b;
        }

        # Function with multiple parameters
        fn calculate_bonus(base: int, multiplier: float) -> float {
        return to_float(base) * multiplier;
    }

    @entry
    fn main() {
    greet("Programmer");
    let sum => add(5, 10);
    let bonus => calculate_bonus(100, 1.5);
    output <- "Sum:", sum, "Bonus:", bonus;
}

Data Structures

You can create custom templates for data using the data keyword. This is perfect for organizing complex information — think of it as creating your own types. A data structure is like a blueprint: it defines what fields exist, their types, and their default values. Then you create instances of that blueprint with specific values. 
            

                player.vice
                
            

            
                data Player {
                name    : string => "Unknown";
                health  : int    => 100;
                pos     : int[]  => [0, 0];
                isAlive : bool   => true;
            }

            @entry
            fn main() {
            # Create instance with custom values
            mut p1: Player => Player(name => "Hero", health => 150);

            # Access and modify fields
            output <- p1.name;           # "Hero"
            p1.health = 140;             # Update health
            p1.pos = [10, 25];           # Move player

            output <- p1.health, p1.pos;
        }
🏗️ Data Structure Features:
• Every field has a default value — no uninitialized data
• Fields can be any type: int, float, string, bool, arrays, even other data structures
• Access fields with dot notation: instance.field
• Create instances with named arguments for clarity

Arrays

Arrays store ordered collections of values of the same type. They are fundamental for working with lists of data — scores, names, positions, anything that comes in multiples. Vice arrays are zero-indexed (the first element is at position 0) and support common operations like push, length checking, and indexed access. 
            

                arrays.vice
                
            

            
                mut values: int[] => [10, 20];    # Create array with 2 elements
                values.push(30);                    # Add element: [10, 20, 30]
                values.push(40);                    # Add another: [10, 20, 30, 40]

                output <- values[0];                # First element: 10
                output <- values[2];                # Third element: 30
                output <- len(values);              # Array length: 4

                # Iterate over array
                for v in values {
                output <- "Value:", v;
            }

UI Engine — Built-in Graphics

One of Vice's most powerful features is the built-in UI engine. Unlike most languages where you need to install external libraries (SDL, SFML, GTK, etc.), Vice has a complete windowing and drawing system integrated directly into the runtime. You can create interactive windows, draw shapes, handle keyboard input, and build simple games — all with zero external dependencies.

The UI engine is designed for simplicity. Creating a window takes one line. Drawing a rectangle takes one line. Checking keyboard input takes one line. This makes Vice perfect for rapid prototyping, game jams, and educational projects where you want to see visual results immediately.

🪟 Window Properties (Colon Syntax)

Vice uses a special colon syntax to check window properties:

win : is_open — Is the window still running?

win : key_up — Up arrow pressed?

win : key_down — Down arrow pressed?

win : key_left — Left arrow pressed?

win : key_right — Right arrow pressed?

            

                ui_demo.vice
                
            

            
                @entry
                fn main() {
                # Create a window
                mut win => ui.create(
                title => "My Vice App",
                size => [800, 600],
                color => "#202020"
            );

            mut x_pos: int => 100;

            # Main game loop
            while (win : is_open) {
            # Handle keyboard input
            if (win : key_right) {
            x_pos += 5;
        }
        if (win : key_left) {
        x_pos -= 5;
    }

    # Draw a colored rectangle
    ui.rect(
    pos => [x_pos, 180],
    size => [50, 50],
    color => "#00ff88"
);

# Draw text
ui.label(
text => "Use arrow keys to move!",

}
}

Advanced "Boss" Lessons

For students and professional developers: see how Vice handles recursion, complex data structures, and real-world patterns like Entity-Component systems.

1. Recursive Factorial

Recursion is when a function calls itself. It's a powerful technique for solving problems that can be broken down into smaller, identical sub-problems. The factorial function is the classic example: fact(5) = 5 × 4 × 3 × 2 × 1 = 120. 

            fn fact(n: int) -> int {
            if (n <= 1) { return 1; }
            return n * fact(n - 1);
        }
        @entry fn main() {
        output <- "Factorial of 5:", fact(5);  # 120
    }

2. Deep Data Stress (ECS Style)

Vice supports nested data structures with clean dot-notation access. This example shows how you can build complex entity systems similar to what game engines use internally. Each Entity contains a Transform, which contains a Vec2 — three levels of nesting, all accessible with simple dots. 

    @autocheck
    data Vec2 { mut x: float => 0.0; mut y: float => 0.0; }
    data Transform { mut pos: Vec2 => Vec2(); }
    data Entity { mut trans: Transform => Transform(); mut id: int => 0; }

    @entry
    fn main() {
    mut player => Entity(id => 1);
    player.trans.pos.x = 100.5;
    player.trans.pos.y = 250.0;
    output <- "Player position:", player.trans.pos.x, player.trans.pos.y;
}

3. Complete Animation Example

Let's put everything together: variables, loops, the UI engine, and conditional logic. This program creates a square that moves across the screen and wraps around when it reaches the edge — a complete, interactive animation in under 30 lines of code.

while (win : is_open) {
x_pos = x_pos + 2;
if (x_pos > 400) {
x_pos = 0;
}
ui.rect(
pos => [x_pos, 180],
size => [40, 40],

}
}

Diagnostics & Error Handling

Vice provides clear, structured error reporting. Every error includes a code number, a human-readable message, and the exact location in your source file where the problem occurred. The error keyword lets you raise errors programmatically with custom codes and messages. 
            

                error_handling.vice
                
            

            
                # Raise a custom error
                error(code => 701) <- "Graphics driver initialization failed!";

                # Conditional error
                if (player_health <= 0) {
                error(code => 404) <- "Player has died — game over.";
            }
🔧 Error Code Ranges:
• 100–199 — Syntax errors
• 200–299 — Type errors
• 300–399 — Runtime errors
• 400–499 — Logic/state errors
• 700–799 — System/IO errors
